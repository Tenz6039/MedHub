## 项目概述

**MedHub** 是一个功能完整的智慧医药电商平台，采用前后端分离的现代化架构设计。系统包含**管理后台**和**用户端小程序**两大核心模块，为医药零售企业提供全方位的数字化解决方案。

### 项目亮点

- **模块化设计**：清晰的三层架构（Common/Pojo/Server），高内聚低耦合
- **安全可靠**：JWT 双令牌认证、微信支付安全、数据加密传输
- **高性能**：Redis 缓存、数据库连接池、分页查询优化
- **多端支持**：Web 管理后台 + 微信小程序用户端
- **用户体验**：Element UI + uni-app，流畅的交互体验
- **开发友好**：Swagger API 文档、统一异常处理、自动填充字段

### 适用场景

- 医药电商平台
- 在线药店系统

## 模块结构说明

### 1. **med-hub-common** - 公共模块

**路径**: `med-hub-common/`

**主要功能**:

该模块是整个项目的公共基础组件库，为其他模块提供通用的技术支持和工具服务。

**核心职责**:

- **常量管理**：统一管理项目中的各类常量定义，包括自动填充规则、优惠券业务常量、JWT 声明常量、消息提示常量、密码规则常量以及状态码常量等，确保全局常量的一致性和可维护性。

- **上下文管理**：提供基于 ThreadLocal 的线程上下文工具，用于在请求处理过程中存储和传递当前用户 ID 等上下文信息，实现线程隔离的数据共享。

- **异常处理体系**：构建了完整的自定义异常类层次结构，涵盖账户异常（锁定、未找到）、优惠券业务异常、药品异常、订单业务异常、用户未登录异常等多种业务场景，配合全局异常处理器实现统一的错误响应。

- **JSON 处理**：配置 Jackson 对象映射器，支持日期格式化、序列化定制等，确保前后端数据交互的格式一致性。

- **配置属性封装**：将阿里云 OSS、JWT、微信支付等第三方服务的配置信息封装为属性类，支持从 YAML 配置文件中读取并自动绑定，便于环境切换和配置管理。

- **统一返回结果**：定义标准化的 API 响应格式，包括通用结果封装和分页结果封装，规范前后端数据交互协议。

- **工具类集合**：提供阿里云 OSS 文件上传、HTTP 客户端请求、JWT 令牌生成与解析、微信支付接口调用等常用工具方法，简化开发流程。

### 2. **med-hub-pojo** - 实体模型模块

**路径**: `med-hub-pojo/`

**主要功能**:

该模块是项目的数据模型定义中心，负责管理所有业务实体和数据传输对象，确保各层之间的数据传递规范和安全。

**核心职责**:

- **DTO（数据传输对象）管理**：定义用于接收前端请求参数的数据传输对象，涵盖分类管理、优惠券管理、员工管理、药品管理、药品组合、订单管理、购物车、用户登录等所有业务场景。DTO 作为 Controller 层的入参载体，负责参数校验和数据初步封装。

- **Entity（实体类）管理**：定义与数据库表一一对应的实体类，包括地址簿、分类、优惠券及使用记录、员工、药品及规格、药品组合及关联关系、订单及明细、店铺、购物车、用户及优惠券等完整业务实体。所有实体类均使用 Lombok 注解简化代码，支持链式构建模式。

- **VO（视图对象）管理**：定义返回给前端的数据视图对象，实现业务数据与展示数据的分离。VO 包含营业数据统计、优惠券统计与详情、员工登录信息、药品及组合详情、订单概览与支付信息、店铺信息、用户登录信息等场景，确保敏感数据（如密码）不会泄露到前端。

- **数据隔离设计**：采用 DTO/Entity/VO 三层分离的设计模式，DTO 用于接收参数，Entity 用于数据库操作，VO 用于返回结果，实现数据的层次化隔离，提高系统安全性和可维护性。

- **Lombok 集成**：所有模型类均使用 Lombok 的 @Data、@Builder、@AllArgsConstructor 等注解，自动生成 getter/setter、构造器、构建者模式等样板代码，大幅简化代码结构。

### 3. **med-hub-server** - 服务层模块

**路径**: `med-hub-server/`

**主要功能**:

该模块是项目的核心业务逻辑实现层，包含完整的 Web 应用架构组件，负责处理所有业务请求、数据持久化和第三方服务集成。

**核心职责**:

- **自定义注解与切面**：提供自动填充注解 @AutoFill 及对应的切面实现，通过 AOP 技术自动处理实体类的创建时间、更新时间、创建人、更新人等审计字段的填充，减少重复代码，确保数据一致性。

- **配置管理**：提供完整的 Spring 配置类，包括阿里云 OSS 对象存储配置、Redis 缓存配置、Spring MVC 配置（拦截器注册、跨域支持、消息转换器）、WebSocket 配置等，支持注解驱动的配置管理。

- **控制器层（Controller）**：分为管理后台接口和用户端接口两大体系。管理后台接口涵盖分类、优惠券、员工、药品、药品组合、订单、报表、店铺、工作台等模块的 CRUD 操作；用户端接口提供地址簿、分类、药品、药品组合、订单、店铺、购物车、用户、优惠券等 C 端业务功能。所有接口均采用 RESTful 风格设计，支持 Swagger/Knife4j 文档生成。

- **全局异常处理**：配置全局异常处理器，统一捕获并处理各类业务异常和系统异常，返回标准化的错误响应格式，提升用户体验和系统可维护性。

- **拦截器体系**：实现 JWT 令牌拦截器，分为管理员令牌拦截器和用户令牌拦截器，分别处理后台管理系统和小程序用户端的身份认证与授权，支持令牌刷新和权限校验。

- **数据访问层（Mapper）**：基于 MyBatis 框架定义数据访问接口，涵盖所有业务实体的 CRUD 操作，配合 XML 映射文件实现复杂 SQL 查询，支持动态 SQL 和结果映射。

- **业务逻辑层（Service）**：定义各业务模块的服务接口，封装复杂的业务逻辑，包括事务管理、缓存处理、业务规则校验、多表关联操作等，确保业务逻辑的内聚性和可测试性。

- **服务实现层（Service Impl）**：实现具体的业务逻辑，集成 Redis 缓存优化查询性能，使用@Transactional 注解管理事务边界，处理并发场景下的数据一致性问题。

- **定时任务**：实现订单超时自动取消任务，基于 Spring Task 定时扫描超时未支付订单并自动取消；实现 WebSocket 心跳检测任务，维持长连接稳定性。

- **工具类**：提供优惠券转换工具、用户优惠券转换工具等业务工具类，封装对象转换和数据处理逻辑。

- **WebSocket 服务**：实现 WebSocket 服务端，支持服务器向客户端推送实时消息，用于订单状态变更通知、系统消息推送等场景。

### 4. **med-hub-admin-vue** - 管理后台前端

**路径**: `med-hub-view/med-hub-admin-vue/`

**技术栈**: Vue 2.6 + TypeScript + Element UI

**主要功能**:

该模块是面向后台管理人员的 Web 前端应用，提供完整的后台管理界面和交互功能，帮助管理员高效管理医药电商平台的各项业务。

**核心职责**:

- **API 接口封装**：基于 Axios 封装统一的 HTTP 请求模块，按业务模块分类管理 API 接口，包括分类、图表统计、优惠券、员工、消息通知、药品、药品组合、订单、用户等所有后端接口的调用，支持请求拦截、响应拦截、错误处理和 Token 自动注入。

- **资源资产管理**：统一管理静态资源文件，包括图标、登录页面图片、404 页面图片、公共图片等，支持图片上传组件和阿里云 OSS 集成。

- **公共组件库**：提供可复用的 UI 组件，包括面包屑导航、图表组件（基于 ECharts 封装）、空状态展示、折叠按钮、头部标签、图片上传组件、输入框自动完成组件等，提升开发效率和界面一致性。

- **图标系统**：基于 SVG 实现图标组件化，提供仪表盘、员工、消息、主要功能、药品、支付、会员等常用图标，支持按需加载和 TypeScript 类型提示。

- **布局系统**：实现经典的后台管理布局，包括顶部导航栏（显示用户信息、消息通知、全屏切换等）、侧边栏菜单（支持多级菜单、图标、权限控制）、主内容区域（支持标签页、面包屑导航），提供响应式适配和侧边栏折叠功能。

- **状态管理（Vuex）**：实现全局状态管理，分为 app 模块（管理侧边栏状态、设备类型等）和 user 模块（管理用户信息、Token、权限等），支持状态持久化和模块化组织。

- **路由系统**：基于 Vue Router 实现前端路由，配置完整的路由表，包括登录页、404 页、工作台、数据统计、订单管理、药品组合管理、药品管理、优惠券管理、分类管理、员工管理等页面，支持路由懒加载、权限控制、动态标题和面包屑。

- **样式体系**：基于 Sass 构建样式系统，包括 Element UI 主题变量定制、图标样式、过渡动画、变量定义、首页样式、侧边栏样式等，支持主题切换和响应式布局。

- **工具函数库**：提供常用工具函数，包括 Cookie 操作（Token 读写、用户信息存储）、日期格式化、数据验证、本地存储等。

- **权限控制**：实现路由级别的权限控制，基于用户角色动态生成可访问的路由表，未登录用户自动跳转到登录页，支持按钮级权限控制。

- **多环境配置**：支持开发环境、测试环境、生产环境的配置切换，包括 API 基础地址、上传地址、环境标识等，支持 PWA 离线访问。

### 5. **med-hub-weixin** - 用户端小程序

**路径**: `med-hub-view/med-hub-weixin/`

**技术栈**: uni-app + Vue 2.x

**主要功能**:

该模块是面向 C 端用户的微信小程序应用，提供完整的药品浏览、购物车、下单支付、订单管理等电商功能，为用户带来便捷的在线购药体验。

**核心职责**:

- **页面体系**：实现完整的电商业务页面，包括首页（展示分类、推荐、搜索）、活动页（促销活动）、地址管理页（增删改查收货地址）、订单详情页（展示订单信息和物流）、历史订单页（订单列表和状态筛选）、药品详情页（规格选择、加入购物车）、购物车页（数量调整、优惠券选择）、下单页（地址选择、优惠券抵扣、备注）、支付页（微信支付）、个人中心页（用户信息、订单入口、优惠券）、我的优惠券页等核心页面。

- **公共组件库**：提供可复用的 UI 组件，包括空状态组件（无数据展示）、图标组件（uni-icons）、导航栏组件（uni-nav-bar）、电话拨打组件、日期选择器组件、弹窗组件（uni-popup）、状态栏组件、底部选择器等，统一界面风格并提升开发效率。

- **uni-app 生态集成**：基于 uni-app 框架实现跨平台能力，集成 uni-ui 组件库，包括徽章（uni-badge）、日历（uni-calendar）、卡片（uni-card）、折叠面板（uni-collapse）、组合选择器（uni-combox）、倒计时（uni-countdown）、数据选择框（uni-data-checkbox）、数据选择器（uni-data-picker）、日期格式化（uni-dateformat）、日期时间选择器（uni-datetime-picker）、抽屉（uni-drawer）、增强输入框（uni-easyinput）等丰富组件。

- **API 接口层**：封装统一的 API 调用模块，处理所有与后端的 HTTP 通信，包括用户登录、药品查询、购物车操作、订单提交、订单查询、地址管理、优惠券领取和使用等接口，支持请求参数序列化、响应数据解析和错误处理。

- **状态管理（Vuex）**：实现全局状态管理，统一管理用户信息、购物车数据、地址信息、订单状态等全局数据，支持跨页面数据共享和状态持久化。

- **样式系统**：基于 Sass 构建通用样式库，包括公共样式、颜色变量、字体大小、间距规范等，确保界面风格一致性；实现响应式布局适配不同屏幕尺寸。

- **静态资源管理**：管理 Tab 栏图标、功能图标、背景图片、默认图片等静态资源，支持图片优化和按需加载。

- **应用配置**：通过 pages.json 配置页面路由、窗口样式、Tab 栏、导航栏样式等；通过 manifest.json 配置小程序 AppID、权限、网络超时等参数；通过 uni.scss 配置全局样式变量。

- **用户体验优化**：实现下拉刷新、上拉加载更多、空状态提示、加载动画、成功/失败提示、确认对话框等交互细节，提升用户体验。

- **微信小程序能力集成**：集成微信小程序原生能力，包括微信登录、微信支付、地址选择器、手机号获取、模板消息等，提供原生体验。
